# Thea Playbook

Ten dokument utrwala kierunek projektu: nauka SwiftUI + MVVM przez realny produkt.

## 1) Mission

- Projekt treningowy, nie "ship at all costs".
- Priorytet: sposob myslenia developerskiego.
- Tempo: male kroki, stale domykanie.

## 2) Role split

- AI: PM + PO + Tech Lead + Senior mentor.
- User: Junior dev implementujacy i uczacy sie przez praktyke.

## 3) Daily rhythm (10-15 min)

Kazda sesja:
- 2 taski praktyczne (male, konkretne, mierzalne).
- 1 pytanie teoretyczne ("dlaczego tak?").

## 4) Definition of done for a task

- Dziala lokalnie bez regresji.
- Logika jest w VM/Model, nie w View.
- Kod czytelny i bez oczywistej duplikacji.
- Krotkie podsumowanie: czego sie nauczyles.

## 5) Product roadmap (MVP+)

### Phase A: Stabilny core
- Domknij jakosc add/edit/delete/search/sort/status.
- Uporzadkuj wspolne komponenty i styl listy.

### Phase B: Fermentation intelligence
- Progress fermentacji.
- ETA (days remaining).
- Status "ready".

### Phase C: Journal quality
- Lepsze notatki batcha i czytelnosc detail view.
- Drobne UX dla szybszego skanowania danych.

### Phase D: Premium calm UI
- Material/translucent cards.
- Delikatna hierarchia kolorow i spacing.
- Spojny design system tokenow (kolory, promienie, spacing).

## 6) Design guardrails

- Styl: calm premium + natural bio.
- Tlo: off-white (#F7F7F5), nie czyste biale.
- Zaokraglenia 16-20pt, brak ciezkich obramowan.
- Delikatna glebia zamiast "krzykliwych" efektow.

## 7) Collaboration protocol

Gdy prosisz o "kolejny dzien":
- dostajesz 2 praktyczne + 1 teoria,
- bez pelnej gotowki kodu na start,
- najpierw pytania naprowadzajace,
- potem feedback i kolejny krok.
